# Rules for Documentation

1. **architecture.md uses `@` references** to domain docs. Do not duplicate domain
   detail in architecture.md — record the decision and point to where detail lives.
2. **todo.md entries need a date.** Format: `- YYYY-MM-DD: description`
3. **history/ is append-only.** Never edit or delete existing history entries.
4. **research/ is reference material.** It informs implementation but is not prescriptive.
